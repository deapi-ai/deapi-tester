# Code Style Rules

- Always TypeScript strict mode
- Never use `any` — define proper types
- React components: functional components with named export
- One component per file
- Hooks in separate files in hooks/
- Use `@/` alias for imports (e.g. `@/lib/storage`)
- Tailwind classes instead of inline styles or CSS modules
- Comments in English
- Error messages in UI in English
- Console.log only with prefix [deapi-tester]
